# 大赛 / 演示用口令序列（降低现场翻车）

以下与 **`config.yaml` → `presets`** 键名一致；群内 **单独一行** 发送（需 @ 机器人，视群配置而定）。

## 路线 A：火星命题（约 3～5 分钟骨架）

1. `帮助` — 确认机器人在线、预设键已加载。  
2. `开始游戏 preset:mars` — 或 `开始游戏 #mars`。  
3. 任选一条推进，例如：`检查氧气读数并与队长同步`。  
4. `游戏状态` — 看幕数与（若未配 Ark）配图跳过累计。  
5. `导出梗概` — 快速分享结构（不结束）。  
6. `结束游戏` — 全文归档 + 累计通关 +1。

## 路线 B：赛博短局（偏风格展示）

1. `开始游戏 #demo`  
2. 推进 1～2 次自由决策。  
3. `导出故事` — 仅导出、不结束（可选）。  
4. `结束游戏`。

## 路线 C：自定义开场（无预设）

1. `开始游戏 一群档案员在雨夜发现一卷无权查阅的磁带`  
2. 正常推进至满意后 `结束游戏`。

## 备忘

- 导出类口令后 **不要** 在同一行跟剧情，否则会被当决策。  
- 单应用仅一条 `event +subscribe`；演示前关掉旧终端里的引擎进程。  
- 未配置 `ARK_API_KEY` 时配图会跳过，**游戏状态** 可查看本局跳过次数。
